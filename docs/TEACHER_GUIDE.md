# 👩‍🏫 Guía del Docente — PASSWORD MASTER

Password Master es una herramienta educativa disfrazada de videojuego. Esta guía explica cómo
integrarla en tu aula.

## Objetivos pedagógicos

1. **Comprender** cómo se construyen contraseñas robustas.
2. **Reconocer** ataques de ingeniería social y phishing.
3. **Aplicar** autenticación multifactor y gestores de contraseñas.
4. **Diferenciar** entre tipos de amenazas y sus mitigaciones.
5. **Reflexionar** sobre higiene digital y cultura de seguridad.

## Cómo integrar la app

### Opción A — Aula invertida (15 min)
1. El estudiante juega 2-3 niveles en casa.
2. En clase, debate con el panel docente abierto.

### Opción B — Laboratorio (45 min)
1. Proyecta el *Modo Laboratorio*.
2. Pide comparar contraseñas de alumnos (anonimizadas) y *visualizar* la entropía.
3. Discute tiempos de *crackeo* a distintas longitudes.

### Opción C — Reto competitivo (60 min)
1. Crea una *clase* desde el panel docente.
2. Comparte el código.
3. Lanza un *Daily Challenge* y revisa el ranking.

## Conceptos cubiertos por nivel

| Rango de niveles | Concepto                                  |
| ---------------- | ----------------------------------------- |
| 1-5              | Tipos de contraseñas y entropía.          |
| 6-10             | Ataques de fuerza bruta.                  |
| 11-15            | Reutilización de contraseñas.             |
| 16-20            | Phishing e ingeniería social.             |
| 21-30            | Gestores y rotación.                      |
| 31-40            | MFA, *biometría*, *passkeys*.             |
| 41-50            | Filtraciones, *zero-day*, *supply-chain*. |

## Métricas del panel docente

- **Progreso medio** (% niveles completados).
- **Conceptos dominados** vs **conceptos a reforzar**.
- **Tiempo medio por sesión**.
- **Tasa de victoria** por nivel (para ajustar dificultad).

## Buenas prácticas para el docente

- **No reveles contraseñas reales** en clase — usa las del laboratorio.
- **Anima a usar gestores** reales (Bitwarden, 1Password, KeePass) tras la sesión.
- **Refuerza MFA** explicando el *por qué* del coste UX vs. ganancia de seguridad.
- **Discute incidentes reales** (filtraciones recientes) para contextualizar.
